## 渲染机制

### 1. 什么事DOCTYPE及作用

DOCTYPE是用来声明文档类型的，告知浏览器的解析器用什么文档标准解析文档 ，XML或是HTML

HTML5: <!DOCTYPE html>

### 2. 浏览器渲染过程

浏览器根据请求的URL，先查询本地的DNS缓存列表，没有则向DNS域名请求解析真实IP，然后向服务器发送请求报文，建立一次TCP连接； 
响应成功后，服务器发送响应报文，浏览器接收文件（HTML、JS、CSS、图象等）； 
浏览器解析响应数据，先创建DOM树，再解析CSS样式，并应用到DOM树，进行渲染，最后下载JS文件进行解析，显示给用户。 

### 3. 重排Reflow

DOM结构中的各个元素都有自己的盒子（模型），当元素添加或删除或发生定位变化，宽度大小变化时，浏览器会将元素放到它该出现的位置。这个过程叫做reflow。

### 4. 重绘Repaint

页面将要呈现的内容，浏览器按照元素各自的特性绘制的过程，叫做repaint。


## JS运行机制

### 1. JS是单线程的

### 2. 任务队列分为同步任务和异步任务

同步任务指的是，在主线程上排队执行的任务，只有前一个任务执行完毕，才能执行后一个任务；
异步任务指的是不进入主线程、而进入任务队列的任务，只有当任务队列通知主进程，某个任务可以执行了，该任务才会进入主线程执行。

### 3. Event Loop

JS的事件分为两种，宏任务和微任务。
宏任务：整体代码script、setTimeout、setInterval、UI rendering
微任务：Promise.then、process.nextTick

宏任务先于微任务执行，当主线程的同步任务完成之后，会从异步队列中读取事件放入主线程执行。主线程从任务队列中读取事件的这个过程是循环不断的，所以整个运行机制又称为Event Loop。

### 4. 哪些语句会放在异步队列中
- setTimeout、setInterval (停顿时间结束之后，才会放入异步队列)
- DOM事件
- Promise
